jQuery SQL Designer

  • jQuery SQL Designer can be accessed at the following URL: http://daas.oucs.ox.ac.uk:8080/jQuerySQLDesigner/designer.seam
  • jQuery SQL Designer is online tool for database designing. It can be used for designing new database, modifying existing database or viewing the design of live database.
  • jQuery SQL Designer is intentionally made "Double Click". This means for each action double click is required. It will be made clear in the remaining instructions.
  • is the Table icon. To start the database designing, drag "Table" icon. This will bring the pop-up suggesting to first create database itself, as shown:

  • Fill the form accordingly by giving the name of database and its description and click button "Create Database". This will result in another pop-up confirming the successful creation of database.
  • Drag the "Table" icon again on the grid to create the table. Create Table pop-up will appear. Please fill the form as required and click the button "Create Table".

  • This will result in the creation of new table with one primary key "ID" as integer. In next versions of the database it will be possible to create a database without any primary key.
  • is the Column icon. It is used to add columns to existing tables.
  • Drag "Column" icon on already created table. This will bring the Create Column pop-up as shown below:

  • There are few things to notice. Fields "Data Length" and "Scale" are either disabled or enabled depending on the Column Type. For data type "char" and "vchar" the field "Data Length" is editable.

  • Similarly for the data type "Decimal" and "Float", fields "Data Length" and "Scale" are editable.
  • If the field is tagged as Primary Key then the form automatically selects the column properties as "Not Null" and "Unique" and made them un-editable.
  • Fill the Create Column form as required and click on the button "Create Column". This will add a new column to the existing table.
  • is the properties icon on the right hand side of Table and Column names. Double clicking on the properties icon brings either Modify Table or Modify Column form. User can verify, modify and update the existing values for the table and columns. The form do also has options to Delete Table or Delete Column buttons.
  • is the minimise icon on the left hand side of the table name. Double click the minimise icon to minimise the table. The table will be minimised and its header will change the background colour and font size as shown below:
  • Tables can be dragged around the design panel by bringing the mouse on table header. Mouse icon changes on entering the table header and changes again when user right click the mouse button.
  • There are various icons for various types of relations i.e. One-To-One (Identifying and Non Identifying), One-To-Many One (Identifying and Non Identifying) and Many-To-Many.
  • When any of the relation button is clicked it changes its appearance as shown below:
    ..... to ......
  • Icons with dashed lines represent Non Identifying relations. Similarly icons with continuous line represent Identifying relations.
  • Drawing relations between two tables involves three steps:
    • Click the appropriate relation icon
    • Double click the header of Table with Primary Key
    • Double click the header of secondary (dependent) Table.
  • Line will be drawn between two tables representing the relation and the relation icon reverts to its normal state.
  • Moving the cursor on the relation will change the colour of the relation and also highlights headers of tables involved.
  • is SQL Generator option and it produces the SQL for the designed database.
  • button loads one existing database. This option is to kick start the process of familiarising with jQuerySQL Designer. It is quite complicated database used by Roman Economy Project in Oxford University.
  • The button "Load Live Database" is a new very powerful feature to draw Design Diagram from the live running database.
  • Click on the "Load Live Database", it will bring the pop-up where enters the appropriate fields. The pop-up form is shown below:
  • We support all major Databases but right now we are only functionality related to MySQL and PostgreSQL is available from GUI.
  • Select the database type; enter the host name where database is located without any protocol i.e. daas.oucs.ox.ac.uk. The following URL http:// daas.oucs.ox.ac.uk will not work.
  • Enter the database port for MySQL the default port is 3306 and for PostgreSQL it is 5432 but contact your system administrator if you are not sure.
  • In the end enter the user name and password of the user who has access to the database.
  • SQL Designer supports the migrating of database from one vendor to another at run time. It follows following steps:
    • Pulls the database design from existing database.
    • Pulls the data from the existing database.
    • Create new database which can even be on different database platform
    • Populate the database with the data.
